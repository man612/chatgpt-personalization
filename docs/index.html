What this builder checks

      
The page uses the repository's public JSON Schema when available. It reports missing fields, wrong value types, unsupported properties, duplicate array items, configured schema limits, and rendered fields that approach or exceed the configured 1,500-character limit.

      
The browser performs structural validation, not the CLI's complete lint pass. Use the Python linter for secret-pattern, duplicate-text, and prompt-bloat checks, then use the manual evaluation guide to compare behavior against a baseline.

      
The three output sections are the project's stable data model. ChatGPT field labels and layouts can vary by product surface, so review the field-mapping guide before pasting them.
