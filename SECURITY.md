# Security and privacy

This repository does not need credentials, API keys, or account exports.

Do not include secrets or sensitive personal data in issues, pull requests, example profiles, or test fixtures. The linter detects a few common credential patterns, but it is not a security scanner and cannot guarantee that a file is safe to publish.

To report a sensitive problem, use GitHub's private vulnerability reporting feature when it is available for this repository. Otherwise, open a minimal issue that describes the affected file without posting the secret or exploit material itself.
